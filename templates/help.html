基础使用说明

通过以下格式访问文件/目录：
http://host:8000/s/虚拟路径?key=你的访问密钥

可添加参数：
- &raw=1 获取纯文本原始内容（仅文件）
- &download=1 下载文件
- &json=1 以 JSON 格式返回结果（适合 AI Agent 解析）

写操作参数（需要对应权限）：
- &mkdir=1 创建目录
- &upload_url=https://... 从 URL 拉取文件到服务器
- &content=文本内容 上传纯文本内容（64KB 上限）
- &delete=1 删除文件或空目录
- &rename_to=新名称 重命名文件/目录
- &move_to=/目标路径 移动到其他目录

其他 HTTP 方法：
PUT /s/{path}?key=xxx — 上传文件（Body 为原始内容）
POST /s/{path}?key=xxx — 上传文件（multipart/form-data）
DELETE /s/{path}?key=xxx — 删除文件或空目录

完整使用说明

本服务为 AI Agent 设计，提供类 FTP 的加密访问。

端点

方法路径说明所需权限
GET/s/{path}?key=<key>查看文件或列出目录read / list
GET/s/{path}?key=<key>&mkdir=1创建目录write
GET/s/{path}?key=<key>&upload_url=URL从 URL 拉取文件write
GET/s/{path}?key=<key>&content=数据上传文本内容（≤64KB）write
GET/s/{path}?key=<key>&delete=1删除文件或空目录delete
GET/s/{path}?key=<key>&rename_to=新名重命名rename
GET/s/{path}?key=<key>&move_to=/目标移动rename
PUT/s/{path}?key=<key>上传文件（Body=原始内容）write
POST/s/{path}?key=<key>上传文件（multipart）write
DELETE/s/{path}?key=<key>删除文件或空目录delete

通用参数

&json=1 — 所有端点返回 JSON 而非 HTML
&raw=1 — 查看文件原始内容（纯文本）
&download=1 — 下载文件

权限说明

每个共享目录可独立配置：list（列出）、read（读取）、write（写入/上传）、delete（删除）、rename（重命名）权限。密钥错误或权限不足返回 403。

安全限制

- 上传文件最大 500MB
- content= 参数最大 64KB
- upload_url 仅支持 HTTPS，拒绝内网地址（SSRF 防护）
- 写操作每分钟最多 60 次（速率限制）
- 文件名自动净化，拒绝路径穿越字符
- 日志中密钥仅记录前4字符

限制 Agent（仅 GET）使用示例

# 创建目录
GET /s/data?key=abc&mkdir=1

# 从 URL 上传文件
GET /s/data?key=abc&upload_url=https://example.com/file.zip

# 上传文本内容
GET /s/data/note.txt?key=abc&content=Hello+World

# 删除文件
GET /s/data/old.txt?key=abc&delete=1

# 重命名
GET /s/data/old.txt?key=abc&rename_to=new.txt

# JSON 输出
GET /s/data?key=abc&json=1

管理

  • /admin — 管理员界面（需管理员密钥）
  • /admin/stats — 服务器统计（JSON）
  • /admin/logs?action=...&path=...&ip=...&keyword=... — 日志搜索
  • /health — 健康检查
